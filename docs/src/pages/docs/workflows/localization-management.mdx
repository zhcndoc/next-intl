一旦你使用 `next-intl` 搭建好了你的应用，你将拥有多个包含消息的翻译包（例如 `en.json`）。为了简化管理这些翻译包的工作流程，并允许其他团队成员贡献翻译，使用本地化管理平台是一个不错的选择。

虽然 `next-intl` 支持所有能够翻译 JSON 文件的本地化管理平台，`next-intl` 推荐使用 <PartnerContentLink name="localization-management-intro" href="https://crowdin.com">Crowdin</PartnerContentLink> 来管理你的翻译。

## 与翻译人员协作

Crowdin 编辑器提供了一个易用的翻译消息环境。除了引导翻译人员完成你的消息外，工作流程还通过机器翻译建议、术语表和上下文截图等高级功能得到了提升。

## 将本地化与开发解耦

作为一个面向开发者的本地化服务，Crowdin 帮助你将本地化流程与开发解耦，并与现有工作流程无缝集成。

**集成选项：**

1. 使用 <PartnerContentLink name="localization-management-developers-integrations" href="https://developer.crowdin.com/cli-tool/">Crowdin 命令行工具（CLI）</PartnerContentLink> 进行命令行集成
2. 通过 <PartnerContentLink name="localization-management-developers-integrations" href="https://support.crowdin.com/github-integration/">GitHub 应用</PartnerContentLink> 进行 Git 集成
3. 通过 <PartnerContentLink name="localization-management-developers-integrations" href="https://developer.crowdin.com/api/v2/#tag/Webhooks">Webhook</PartnerContentLink> 触发自动工作流程
4. 通过 <PartnerContentLink name="localization-management-developers-integrations" href="https://developer.crowdin.com/sdk-ota-web/">JS SDK</PartnerContentLink> 实现无线更新（OTA）交付
5. 手动上传和下载消息

## 使用 GitHub 集成的示例工作流程

本示例使用一个在 GitHub 上公开的示例仓库：[Street Photography Viewer](https://github.com/amannn/street-photography-viewer)。这是一个 Next.js 应用，展示来自 Unsplash 的街头摄影照片，并使用 `next-intl` 进行所有国际化需求。

一旦你有了包含你应用的 GitHub 仓库，可以按照以下步骤操作：

1. 安装 <PartnerContentLink name="localization-management-example" href="https://store.crowdin.com/github">Crowdin 商店中的 GitHub 应用</PartnerContentLink>
2. 按照 <PartnerContentLink name="localization-management-example" href="https://support.crowdin.com/github-integration/">GitHub 集成的设置指南</PartnerContentLink> 进行配置

完成此流程后，Crowdin 将根据你的设置向你的仓库提交一个配置文件。

```yml filename="crowdin.yml"
files:
  - source: /messages/en.json
    translation: /messages/%locale%.json
```

该配置文件向 Crowdin 提供了仓库中的本地翻译文件。
