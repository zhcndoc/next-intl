# 验证消息

为了确保消息的质量和完整性，您可以使用 [i18n-check](https://github.com/lingualdev/i18n-check)。

这个 CLI 可帮助您验证以下问题：

1. 目标语言中缺失的翻译
2. 翻译中 ICU 参数使用不一致

**用法：**

```bash
npx @lingual/i18n-check@latest --source en --locales messages
```

**输出：**

```
发现缺失的键！
┌────────────────────┬───────────────────────────────┐
│ 文件               │ 键                           │
├────────────────────┼───────────────────────────────┤
│  messages/de.json  │  NewsArticle.title            │
└────────────────────┴───────────────────────────────┘
```

<Callout>

您还可以使用 [`--unused`](https://github.com/lingualdev/i18n-check#--unused--u) 标志检测定义在消息中的键，但在代码库中未被引用的键（目前为实验功能）。

</Callout>
